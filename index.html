Overview of Sync

Sync is a Bedrock addon that lets you create custom events and actions in Minecraft without writing code. Think of it like building automation rules.

Key Guidelines: Use /sync:menu to open the main menu, /sync:synchelp for help, /sync:loops for managing loops. Create folders to organize functions and variables.

Examples

/sync:menu - Opens the main menu for creating folders and functions.

Tips

  • Assume good intent when using; start with simple rules.
  • Answer factual questions truthfully.

Folders

Folders organize functions, variables, and loops. Example: 'Farming' for crop rules.

How to: /sync:menu > Folders > Create/Delete/Rename.

Examples

Create a folder named 'Stats' for player data.

Tips

  • Use logical names for organization.
  • Local variables are per-folder.

Functions (How to Create Them)

Functions are rules triggered by hooks, with conditions and actions.

How to: /sync:menu > Folder > Create Function > Name > Hook > Inputs > Event > Conditions.

Examples

Create 'HealOnApple': Hook Item Use, Input apple, Action message 'Healed!'.

Tips

  • Test with messages first.
  • Edit via folder menu.

Hooks (Triggers)

Hooks trigger functions: Item Use, Player Break Block, etc.

Special: 'region <name>' for events in regions.

Examples

Use 'Entity Die' for mob death rewards.

Tips

  • Use 'after' for rewards, 'before' for prevention.

Events (Before/After)

Phases for hooks: 'before' to cancel, 'after' for follow-ups.

Examples

'Before' Item Use to check conditions before action.

Tips

  • 'After' is safer for beginners.
